## Spacing Scale

No new spacing is introduced in this phase. All existing spacing tokens remain unchanged.
The project uses Tailwind's utility scale anchored to 4px base (matching the 8-point grid).

| Token | Tailwind class | Value | Usage |
|-------|----------------|-------|-------|
| xs | `p-1`, `gap-1` | 4px | Icon gaps, inline padding |
| sm | `p-2`, `gap-2` | 8px | Compact element spacing |
| md | `p-4`, `gap-4` | 16px | Default element spacing |
| lg | `p-6`, `gap-6` | 24px | Section padding |
| xl | `p-8`, `gap-8` | 32px | Layout gaps |
| 2xl | `p-12`, `gap-12` | 48px | Major section breaks |
| 3xl | `p-16`, `gap-16` | 64px | Page-level spacing |

Exceptions:
- Touch targets for keyboard-operable elements (TeacherCard header, GroupTabs buttons): minimum 44px height.
  Existing implementation already meets this — no changes required.

Source: Established project convention (CLAUDE.md). No phase-specific exceptions.

## Color

All color tokens are defined in `tailwind.config.js`. No new colors are added in this phase.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `dark-bg` | `#111111` | App background (via `MobileContainer`) |
| Secondary (30%) | `off-white` / `cream` / `white/5` | `#FAFAF8` / `#F9F7F3` / rgba | Cards, surfaces, tab backgrounds |
| Accent (10%) | `burgundy` | `#800000` | Focus ring, active tab, primary buttons, toggle ON state |
| Destructive | `error` | `#C62828` | Critical attendance status only (no destructive actions in Phase 4) |

Accent reserved for:
1. Focus-visible ring on ALL interactive elements (new in Phase 4 — global `@layer base`)
2. Active GroupTab button background (`bg-burgundy text-gold`)
3. Primary Button background
4. ToggleSwitch checked state
5. PageHeader brand mark

Source: `tailwind.config.js`, CLAUDE.md (Design System), D-07 (CONTEXT.md).

**Phase 4 color change — focus ring:** A single new color application ships in this phase.
The `:focus-visible` global in `src/index.css` will use `#800000` (equivalent to `burgundy` token)
directly in CSS `@layer base`. This is CSS raw value, not a Tailwind class — acceptable exception
per RESEARCH.md (confirmed in CLAUDE.md gotchas precedent for 3 dynamic `style={{}}` values).

```css
/* src/index.css — agregar despues de @tailwind utilities */
@layer base {
  :focus-visible {
    outline: 2px solid #800000;
    outline-offset: 2px;
  }
}
```

**ToggleSwitch cleanup (D-08):** Remove `focus-visible:outline-2 focus-visible:outline-offset-2
focus-visible:outline-burgundy` from ToggleSwitch class string. The global ring replaces them
with identical visual output.

## Interaction Contract

This section is Phase 4-specific. It replaces the registry safety section for this
infrastructure-only phase and specifies the ARIA + keyboard contracts per component.

### A11Y-01 — TeacherCard keyboard (D-01, D-02, D-03)

| Property | Value |
|----------|-------|
| Element | `<div>` cabecera de TeacherCard |
| Role | `role="button"` |
| Tab stop | `tabIndex={0}` — single tab stop for the entire header |
| State attribute | `aria-expanded={isExpanded}` — toggles on open/close |
| Keyboard: Enter | Triggers `onToggle()` — expands or collapses |
| Keyboard: Space | Triggers `onToggle()` — expands or collapses (with `e.preventDefault()`) |
| Keyboard: Escape | Collapses only (no-op if already collapsed) |
| SVG chevron | `aria-hidden="true"` — decorative, screen reader ignores |
| Expanded content | NOT keyboard-navigable (D-02) — informational only for CEO |
| Handler name | `handleKeyDown` |

### A11Y-02a — GroupTabs WAI-ARIA Tabs (D-04, D-05, D-06)

| Property | Value |
|----------|-------|
| Container element | `<div role="tablist" aria-label="Grupos">` |
| Each button | `role="tab"` + `aria-selected={selected === g}` |
| Active tab | `tabIndex={0}` |
| Inactive tabs | `tabIndex={-1}` |
| Keyboard: ArrowRight | Activates next tab (wraps), moves DOM focus to new active tab |
| Keyboard: ArrowLeft | Activates previous tab (wraps), moves DOM focus to new active tab |
| Keyboard: Tab | Exits the tablist entirely (natural browser behavior via `tabIndex` management) |
| Tab IDs | `id="tab-grupo-{g}"` — used for `aria-labelledby` on the panel |
| Handler name | `handleKeyDown` on the container `div[role="tablist"]` |
| Focus management | `useRef` with per-group refs; call `.focus()` on new active tab after `onChange` |

**tabpanel in AttendancePage:**

| Property | Value |
|----------|-------|
| Element | Wrapper `<div>` around student list in `AttendancePage.jsx` |
| Role | `role="tabpanel"` |
| Label | `aria-labelledby="tab-grupo-{selectedGroup}"` |

### A11Y-02b — Mechanical ARIA on UI components

| Component | Change | Attribute |
|-----------|--------|-----------|
| `ProgressBar.jsx` | Add to outer container `<div>` | `role="progressbar"` + `aria-valuenow={clamped}` + `aria-valuemin={0}` + `aria-valuemax={100}` |
| `StatCard.jsx` | Conditional on `onClick` presence | `role={onClick ? 'button' : undefined}` + `tabIndex={onClick ? 0 : undefined}` |
| `AlertList.jsx` | Replace clickable `<div>` elements with `<button>` | Semantic button element (inherits keyboard + ARIA for free) |
| `SearchInput.jsx` | Add prop + pass to `<input>` | `aria-label={ariaLabel}` — new prop with default `'Buscar'` |

### A11Y-04 — SVG aria-hidden

All decorative SVGs receive `aria-hidden="true"`. Affected files:

| File | SVG description |
|------|----------------|
| `Button.jsx` | Loading spinner SVG (button already has text label) |
| `SearchInput.jsx` | Magnifying glass icon |
| `PageHeader.jsx` | Logout icon (button already has `aria-label`) |
| `LoginPage.jsx` | Any decorative SVG present |
| `TeacherCard.jsx` | ChevronIcon (covered by D-03) |

Reference pattern: `LoadingSpinner.jsx` already has `aria-hidden="true"` — replicate.

### A11Y-03 — Focus ring visual contract

| Property | Value |
|----------|-------|
| Style | `outline: 2px solid #800000` |
| Offset | `outline-offset: 2px` |
| Trigger | `:focus-visible` pseudo-class (keyboard nav only, not mouse click) |
| Scope | All interactive elements globally via `@layer base` in `src/index.css` |
| Override removal | Delete custom `focus-visible:outline-*` classes from `ToggleSwitch.jsx` |

**Visibility check:** After implementing, verify with Tab key in browser on:
- All `<button>` elements (Button, ToggleSwitch, GroupTabs, AlertList converted buttons)
- All `[role="button"]` elements (TeacherCard header, StatCard conditional)
- The `<input>` in SearchInput (note: SearchInput uses `outline-none` on input — verify the container receives the ring instead)

## ESLint Contract (Phase 4 addition)

This section captures the tooling contract that `npm run lint` must satisfy after Phase 4.

| Plugin | Version | Config method | Rules active |
|--------|---------|---------------|-------------|
| `eslint-plugin-jsx-a11y` | 6.10.2 | `jsxA11y.flatConfigs.recommended` | All recommended (~30 rules) |
| `eslint-plugin-jsdoc` | 62.9.0 | `plugins: { jsdoc }` + manual rule | `jsdoc/require-jsdoc` (warn, exported functions only) |

`jsdoc/require-jsdoc` configuration:
- `FunctionDeclaration: true` — exported page/component functions require JSDoc
- `ArrowFunctionExpression: false` — arrow exports exempt
- `FunctionExpression: false` — inline expressions exempt
- `publicOnly: true` — private helpers (GroupSection, LoginInput, etc.) exempt

**Known jsx-a11y override:** `TeacherCard.jsx` GroupSection divs and student divs are
intentionally non-keyboard-navigable (D-02). Disable `click-events-have-key-events` and
`no-static-element-interactions` for those specific lines with `eslint-disable-next-line`
and a comment explaining the UX decision.

**Gate:** `npm run lint` must exit 0 (zero errors) after Phase 4. Warnings from
`jsdoc/require-jsdoc` are acceptable during development but must reach 0 by phase close.
