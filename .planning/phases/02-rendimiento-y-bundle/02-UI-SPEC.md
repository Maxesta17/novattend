# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Rendimiento y Bundle.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Scope note:** Esta fase no introduce nuevas pantallas ni refactors de layout.
> Los dos unicos elementos UI nuevos son el banner de actualizacion del Service Worker
> y el componente LoadingSpinner para el fallback de Suspense. Todo lo demas
> (memoizacion, debounce, manualChunks, Promise.all) es implementacion mecanica sin
> impacto visual.

## Nuevos Componentes — Contrato de Implementacion

### 1. UpdateBanner (SW update notification)

**Archivo:** `src/components/ui/UpdateBanner.jsx`
**Patron de referencia:** `src/components/ui/ErrorBanner.jsx`

| Propiedad | Valor |
|-----------|-------|
| Posicion | `fixed bottom-0 left-0 right-0 z-50` — barra fija en la parte inferior |
| Contenedor externo | `max-w-[430px] mx-auto` — respeta el frame mobile de MobileContainer |
| Fondo | `bg-gold-soft` |
| Borde | `border-t border-gold/40` |
| Padding | `px-4 py-3` |
| Layout | `flex items-center justify-between gap-3` |
| Visibilidad | Siempre visible cuando `needRefresh === true`. Sin boton de cierre (D-03). |
| Texto | "Nueva version disponible" — `text-text-dark text-sm font-montserrat` |
| Boton "Actualizar" | `bg-burgundy text-white text-xs font-bold font-montserrat rounded-lg px-4 py-[11px] shrink-0` |
| Hover boton | `hover:bg-burgundy-light transition-colors duration-200` |
| Role ARIA | `role="status"` en el contenedor (no es error, es informativo) |
| aria-live | `aria-live="polite"` — no interrumpe lector de pantalla mid-task |

**Comportamiento:** El componente recibe `needRefresh: boolean` y `onUpdate: function`.
Renderiza null cuando `needRefresh === false`. No tiene estado interno.

### 2. LoadingSpinner (Suspense fallback)

**Archivo:** `src/components/ui/LoadingSpinner.jsx`
**Patron de referencia:** SVG spinner ya existente en `Button.jsx` (animate-spin)

| Propiedad | Valor |
|-----------|-------|
| Contenedor | `min-h-screen bg-dark-bg flex flex-col items-center justify-center gap-3` |
| Icono SVG | Circulo de 32x32px con arco animado — `animate-spin` de Tailwind |
| Color circulo base | `stroke-burgundy/30` (circulo completo, opacidad baja) |
| Color arco animado | `stroke-gold` (arco de 75%, strokeLinecap: round) |
| Texto "Cargando..." | `text-white/80 text-sm font-montserrat font-normal` |
| Duracion animacion | 0.8s — definida via `animation: spin 0.8s linear infinite` en Tailwind |

**Nota sobre animacion:** Tailwind `animate-spin` usa `spin` keyframe ya incluido en Tailwind core
(360deg en 1s). Para 0.8s se usa clase custom `[animation-duration:0.8s]` o se agrega
`spinner` en `src/styles/animations.css` si la duracion exacta importa. Claude tiene
discrecion sobre este detalle — `animate-spin` por defecto es aceptable.

## Interacciones y Estados

### SW Update Banner

| Estado | Descripcion |
|--------|-------------|
| Oculto (default) | `needRefresh === false` — el componente devuelve `null`. Sin espacio en layout. |
| Visible | `needRefresh === true` — barra aparece en la parte inferior. No anima la entrada (simplicidad). |
| Loading (post-click) | El boton muestra el spinner SVG de Button.jsx (animate-spin 16x16) y se deshabilita mientras el SW activa. Timeout de 5s por si falla. |
| Post-actualizacion | El SW recarga la pagina automaticamente — el banner desaparece con la recarga. |

### LoadingSpinner (Suspense fallback)

| Estado | Descripcion |
|--------|-------------|
| Activo | Pantalla completa `min-h-screen bg-dark-bg`. Spinner rotando + "Cargando..." |
| Duracion esperada | < 500ms en conexion normal (lazy load de chunk). Si dura mas, es senal de problema de red. |
| No tiene estado de error | Si el chunk falla, React lanza al ErrorBoundary global — no al spinner. |

### Debounce (PERF-03) — sin impacto visual

El campo `SearchInput` existente en DashboardPage no cambia visualmente.
El debounce de 300ms es completamente silencioso (D-08 de CONTEXT.md).
No se agrega ninguna indicacion de "buscando..." ni spinner en el campo.
